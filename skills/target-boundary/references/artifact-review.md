# 已有目标边界合同复核手册

## 目标

- 判断已有 `boundary.md` 是否仍是可执行的目标边界约束。
- 判断后续 agent 是否可以依据合同进入实现设计或实现。
- 本阶段不更新目标边界合同。
- 本阶段不重新生成边界结论。

## 执行规则

### 合同定位

- 只复核 `.codeartz/<topic>/boundary.md`。
- 同一 `<topic>` 只允许一个有效合同。
- 找不到合同时，报告缺失路径。
- 存在多个有效合同候选时，停止开工判断。
- 多合同冲突需要用户裁定唯一有效合同。

### 最新授权

- 授权变化检查
  - 检查项：合同生成后，用户是否给出改变范围、行为或取舍的明确决定。
  - 放行条件：没有更新的明确决定。
  - 失败条件：最新用户决定与合同不一致。
  - 处理方式：判定合同失效，并要求先更新合同。

### 合同完整性

- 必需章节检查
  - 检查项：是否包含边界结论、证据基线、待裁决、目标合同、分区授权、目标边界、备选与被拒边界、风险、依据与系统现实、开工入口。
  - 放行条件：全部必需章节存在。
  - 失败条件：任一必需章节缺失。
  - 处理方式：停止开工判断，并要求先更新合同。

- 目标合同字段检查
  - 检查项：目标合同是否包含目标行为、输入、输出、状态变化、副作用、失败行为、用户可见变化和兼容约束。
  - 放行条件：八个合同项均已填写。
  - 失败条件：任一合同项缺失、留空或保留模板占位符。
  - 处理方式：停止开工判断，并列出缺失合同项。

- 分区授权检查
  - 检查项：分区授权矩阵是否覆盖全部已知分区。
  - 检查项：每个分区是否包含授权状态、目标行为、证据、处理方式和越界后果。
  - 放行条件：分区授权矩阵完整，且授权状态只使用 `SKILL.md` 规定的值。
  - 失败条件：已知分区缺失、字段缺失或授权状态无效。
  - 处理方式：停止开工判断，并列出缺失或无效分区。

- 结构内容检查
  - 检查项：关键结论、证据索引和开工入口是否存在可执行内容。
  - 放行条件：适用字段已有内容，不适用字段明确写“不适用”。
  - 失败条件：适用字段留空、保留模板占位符或以普通风险替代未授权分区。
  - 处理方式：停止开工判断，并列出损坏内容。

### 证据基线

- 基线完整性检查
  - 检查项：合同是否记录仓库、分支、HEAD、工作区状态、相关依赖和分析时间。
  - 放行条件：适用字段均有值。
  - 失败条件：适用字段缺失、为空、写成“未知”或无法解释。
  - 处理方式：判定证据基线失效，并报告缺失字段。

- 仓库基线检查
  - 检查项：当前仓库、分支、HEAD 和工作区状态是否与合同基线兼容。
  - 放行条件：差异不会改变合同中的当前行为、边界或关键证据。
  - 失败条件：差异可能改变当前行为、边界或关键证据。
  - 处理方式：判定合同失效，并列出差异。

- 依赖基线检查
  - 检查项：合同记录的相关锁文件、版本或指纹是否仍然一致。
  - 放行条件：没有相关依赖变化，或变化已证明不影响合同。
  - 失败条件：相关依赖变化可能改变外部约束或推荐结论。
  - 处理方式：判定合同失效，并列出待复核依赖。

### 关键证据

- 证据定位检查
  - 检查项：“证据索引”和“先读”中的文件、符号、测试、配置或设计资料是否仍可定位。
  - 放行条件：关键证据均可定位。
  - 失败条件：任一关键证据缺失、改名或不可访问。
  - 处理方式：判定合同失效，并列出失效证据。

- 证据语义检查
  - 检查项：关键证据是否仍支持合同记录的当前行为和分区边界。
  - 放行条件：证据语义没有改变。
  - 失败条件：证据语义已经改变或出现新反证。
  - 处理方式：判定合同失效，并说明受影响结论。

### 合同状态

- 状态检查
  - 检查项：合同状态是否为“已采纳”。
  - 放行条件：合同状态为“已采纳”。
  - 失败条件：合同状态缺失、保留占位符，或不是“已采纳”。
  - 处理方式：停止开工判断，并报告仍需完成的状态条件。

- 待裁决检查
  - 检查项：合同是否仍有未处理的待裁决项。
  - 放行条件：待裁决为“不适用”。
  - 失败条件：仍有待裁决项，或状态与待裁决内容矛盾。
  - 处理方式：停止开工判断，并列出待裁决项。

### 开工入口

- 先复核检查
  - 检查项：“先复核”中的系统事实、边界和外部约束是否已经重新确认。
  - 放行条件：全部复核通过。
  - 失败条件：存在未复核项或复核失败项。
  - 处理方式：停止开工判断，并列出未通过项。

- 先补检查
  - 检查项：“先补”是否为“不适用”。
  - 放行条件：没有实现前必须补齐的证据。
  - 失败条件：仍有必须补齐的代码事实或外部事实。
  - 处理方式：停止开工判断，并列出待补内容。

- 过期触发检查
  - 检查项：“过期触发”列出的用户决定、需求、代码、分支、依赖、配置、数据或设计资料是否发生变化。
  - 放行条件：没有命中过期触发。
  - 失败条件：任一过期触发已命中。
  - 处理方式：判定合同失效，并说明受影响边界。

### 开工裁定

- 开工放行
  - 放行条件：最新授权、合同完整性、证据基线、关键证据、合同状态和开工入口检查全部通过。
  - 处理方式：说明可以进入实现设计或实现。
  - 禁止：不把开工放行说成已经实现、验证或可上线。

- 开工拒绝
  - 失败条件：任一检查未通过。
  - 处理方式：说明不能依据当前合同开工。
  - 处理方式：列出失效证据、待补内容或待裁决项。
  - 禁止：不在本阶段静默修订合同。

## 输出要求

- 输出开工结论。
- 输出合同路径与合同状态。
- 输出证据基线比较结果。
- 输出最先复核的关键证据编号和定位。
- 未放行时输出阻塞原因和下一步动作。
- 不输出完整探索链路。
