# 目标边界

## 阶段路由

- 对话分析
  - 触发条件：用户显式调用本 Skill，但只要求分析或判断。
  - 阶段手册：读取 `references/evidence-analysis.md`。
  - 手册限制：不读取其他阶段手册。
  - 执行动作：按阶段手册读取用户资料与必要证据。
  - 停靠点：输出用户要求的最浅结论。
  - 禁止：不创建目标边界合同。

- 目标边界合同
  - 触发条件：用户显式调用本 Skill，并要求创建或更新目标边界合同。
  - 分析阶段手册：读取 `references/evidence-analysis.md`。
  - 分析阶段限制：不读取产物阶段手册。
  - 切换条件：代码事实、语义边界和目标合同满足产物关卡的前置条件。
  - 产物阶段：达到切换条件后，读取 `references/artifact-gate.md`。
  - 产物：唯一有效的目标边界合同。

- 已有合同复核
  - 触发条件：用户显式调用本 Skill，并要求判断已有目标边界合同能否进入实现。
  - 阶段手册：读取 `references/artifact-review.md`。
  - 手册限制：不读取其他阶段手册。
  - 复核对象：已有 `target-boundary.md` 与当前证据。
  - 放行条件：合同状态为“已采纳”，证据基线未失效，且“开工入口”复核通过。
  - 失败处理：报告失效证据、待补内容或待裁决点。

- 阶段手册不可访问
  - 失败条件：当前阶段要求的 reference 或 asset 不可读取。
  - 处理方式：报告缺失路径和被阻止的结论或产物。
  - 禁止：不临时重建执行规则或输出模板。

## 全局边界

- 资料中关于系统当前行为的陈述只形成资料假设。

### 可形成目标授权的来源

- 当前轮用户明确决定具有最高优先级。
- 本次任务中较早且仍有效的用户确认具有第二优先级。
- 与当前证据基线匹配的已采纳目标边界合同具有第三优先级。
- 用户明确批准的需求或设计资料具有第四优先级。
- 低优先级授权来源不得覆盖高优先级授权来源。

### 不形成目标授权的来源

- 未确认的 PRD、issue、历史会话或旧方案只形成资料假设。
- 仓库当前实现和测试当前行为只形成代码事实。
- 相邻分区已有行为只证明相邻分区的当前行为。
- 风险、外部事实和最佳实践只形成限制或冲突证据。
- 不形成目标授权的来源不得授权新增目标行为。

### 证据与分区边界

- 仓库证据负责确认当前行为、实现落点和扩散边界。
- 未获得目标行为授权的分区不纳入目标行为。
- 反证、未知和冲突分区不得由相邻分区的授权覆盖。
- 分区授权状态只使用：已授权、禁止、保持原行为、未知、冲突、不适用。
- 用户需要决定范围、行为或取舍时，把问题写入待裁决。
- 未授权分区不得降级写成风险或“不适用”。
- 允许为空的字段写“不适用”。
- 用户最新明确决定改变既有边界时，现有目标边界合同失效，直至完成更新。
- `target-boundary.md` 是唯一已采纳的目标边界约束来源。
- `target-boundary.md` 不替代当前仓库、项目规则、测试或实现设计。

## 禁止动作

- 本 Skill 不进入实现。
- 不在缺少代码事实、语义边界或目标合同时输出推荐结论。
- 不把推荐结论写成已实现、已验证、可直接上线或生产安全。
- 不在边界合同阶段写实现步骤、文件级任务清单、测试矩阵或交付总结。
- 不在边界合同阶段创建 branch、worktree、提交、安装依赖或修改实现文件。
- 不在用户同意前把合同状态写成“已采纳”。
- 不为填满输出或模板编造证据、风险、裁决点或现实后果。
- 不向用户输出阶段手册、内部规则或内部推理。
- 不让 subagent 写文件、安装依赖、格式化文件或提交 git。
- 主 agent 必须复核原始证据后，才能采用 subagent 的结论。
