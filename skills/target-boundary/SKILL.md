# 目标边界

## 阶段路由

- 对话分析
  - 触发条件：用户显式调用 `$target-boundary`，但只要求分析或判断。
  - 执行动作：读取 `references/target-boundary.md`。
  - 停靠点：输出用户要求的最浅结论。
  - 禁止：不创建方案文件。

- 方案文件
  - 触发条件：用户显式调用 `$target-boundary`，并要求创建或更新目标边界方案。
  - 执行动作：读取 `references/target-boundary.md`。
  - 放行条件：代码事实、语义边界和目标合同已满足方案写入关卡。
  - 产物：`.codeartz/<topic>/target-boundary.md`。

- 已有方案复核
  - 触发条件：用户显式调用 `$target-boundary`，并要求判断已有方案能否进入实现。
  - 执行动作：读取已有 `target-boundary.md`。
  - 放行条件：方案状态为“已采纳”，且“开工入口”复核通过。
  - 失败处理：报告失效证据、待补内容或待裁决点。

- 手册不可访问
  - 失败条件：`references/target-boundary.md` 不可读取。
  - 处理方式：报告缺失路径和被阻止的结论或产物。
  - 禁止：不临时重建执行规则或输出模板。

## 全局边界

- 需求、PRD、issue、会话和设计资料只形成资料假设。
- 当前系统行为、系统现实和根因判断必须由仓库证据支撑。
- 未获得用户或仓库证据授权的分区不纳入目标行为。
- 反证、未知和冲突分区不得由相邻分区的授权覆盖。
- 风险和外部事实只约束方案可行性和方案限制。
- 风险和外部事实不得授权新增目标行为。
- 用户需要决定范围、行为或取舍时，把问题写入待裁决。
- 方案文件是后续 agent 接手时的唯一方案来源。

## 禁止动作

- 不在缺少代码事实、语义边界或目标合同时输出推荐方案。
- 不把推荐方案写成已实现、已验证、可直接上线或生产安全。
- 不在方案阶段写实现步骤、文件级任务清单、测试矩阵或交付总结。
- 不在方案阶段创建 branch、worktree、提交、安装依赖或修改实现文件。
- 不在用户同意方案前把方案状态写成“已采纳”。
- 不让 subagent 写文件、安装依赖、格式化文件或提交 git。
- 主 agent 必须复核原始证据后，才能采用 subagent 的结论。
