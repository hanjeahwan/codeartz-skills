# Agent Evolve

## 触发条件

- 上下文存在 `AGENT EVOLVE ACTIVE — mode: safe` 或 `AGENT EVOLVE ACTIVE — mode: review` 时，判断当前用户消息或当前任务证据是否形成可复用候选。
- 命中以下任一候选时，自动运行本技能：
  - 用户明确要求把具体规则写入项目沉淀或未来 agent 会读取的规则源。
  - 用户直接给出或修正能判断何时适用、会怎样改变未来 Agent 选择的规则。
  - 当前任务证据形成有依据、可执行且作用域可收敛的未来决策候选。
  - 当前任务输入或证据与有效项目规则发生会改变实现方向的冲突。
  - 当前任务或连续纠正中出现可独立成立的通用原则。
  - 多轮反馈形成精确候选，但仍需确认合并关系、验证状态或冲突裁决。
- 先判断排除项。只有当前事实、单次成功、一次性操作边界、当前产物细节、空泛目标、不能推出可复用决策的失败、无独立证据的 Agent 观察、用户明确禁止泛化、没有具体原则的普通否定，或当前任务只是执行现有规则且用户与任务证据都没有提出规则候选时，不自动运行本技能；没有另一条独立候选时立即停止触发判断，禁止读取工作流或安全验证。用户直接提出的精确规则即使可能已被现有规则覆盖，仍进入查重判断。
- 用户手动调用 `$agent-evolve` 时运行本技能。
- 发送当前任务的最终回复前，必须判断本轮是否作出了由用户权限或任务证据已经确定、未来会重现的 A→B 选择；是则加载，不要求先发生故障或用户提及规则。Agent 为当前任务自行提出的建议、备选或待确认合同不算已经确定的选择。
- 空泛目标缺少用户或任务证据直接支持的适用条件与决策后果时禁止自动加载；不要先加载再替用户细化规则，也不要起草规则文本或邀请用户批准持久化。
- 使用完整语义判断候选；“以后”“必须”“沉淀”等表达只能作为证据，不能单独决定触发。

## 模式路由

- `safe`（安全沉淀）：读取相对 `references/workflow.md`，进入安全沉淀模式。
- `review`（审核后沉淀）：读取相对 `references/workflow.md`，进入审核后沉淀模式。
- 进入工作流时禁止预读 `references/validation.md`。
- 工作流明确进入安全验证阶段时，读取相对 `references/validation.md`。
- 用户手动调用 `$agent-evolve`，且当前上下文没有 `AGENT EVOLVE ACTIVE — mode: safe|review` 时，进入关闭模式下的手动调用。
- 关闭模式下的手动调用读取相对 `references/workflow.md`。

## 全局边界

- 只在与用户对话的主会话自动处理。
- 普通任务开始时不预读本技能；任务执行中出现合格证据时，在同一轮加载并处理。
- 先继续完成当前用户任务，再在同一轮完成反馈决策。
- 当前任务与有效项目规则发生未裁决冲突时，先停止会选定冲突方向的实现或测试修改，再进入提案与裁决。

## 禁止动作

- 禁止扫描完整对话记录做会后复盘。
- 禁止使用模型记忆补造反馈。
- 禁止创建反馈收集箱。
- 禁止自动提交 git commit。
- 禁止启动额外 agent 轮次处理反馈。
