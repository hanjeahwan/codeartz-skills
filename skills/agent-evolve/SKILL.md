# Agent Evolve

把会随当前会话丢失、但会改变未来 Agent 决策的判断转成当前项目规则。

## 入口

- 自动入口：存在 `AGENT EVOLVE ACTIVE — mode: safe|review` 时，逐项检查任务过程和最终回复前出现的独立未来决策。
- 自动候选前置：先从直接证据写出 `稳定判据 → 决策后果 @ 未来作用域`。实例定位、执行或验收结果、待执行动作，以及“以后注意”之类警告不能提供稳定判据或未来作用域。删除这些内容后任一项为空，或删除候选后动作不变时，立即停止且不读取 reference。
- 候选条件：候选必须有可靠依据和未来差异，且不是单纯遵循或修改当前有效权威已经负责的同一决定。禁止从当前动作反推适用条件或作用域。
- 手动入口：用户调用或点名 Agent Evolve、要求项目沉淀、评估反馈或审核精确候选时，直接读取 `references/workflow.md`。
- 手动入口的作用域、重复、冲突和结果由工作流判断。禁止因候选越界、重复、未决或用户暂不写入而在入口返回。
- 交付分离：关闭 Agent Evolve 后，当前任务本来仍须交付的条件、后果和作用域只算任务交付。目标文件、写入路径或是否修改规则源不同不能证明独立。
- 所有权排除：当前任务或领域的有效权威已经明确负责同一条件、后果和作用域时，遵循或按任务修改原权威内容属于当前交付；执行偏差不能证明缺少项目规则，也不得把决定复制到另一规则源。当前指令只拥有本次交付范围，用户明确给出的额外未来作用域仍可形成候选；现有长期规则的同义覆盖由工作流返回已有规则覆盖。
- 独立候选：同一依据还能指导交付范围外的未来任务时，只处理有依据的额外条件、后果或作用域。
- 自动排除：当前有效权威内的遵循、执行纠正或原位修改，以及当前事实、局部细节、一次性要求、单次成功、普通否定、空泛目标、无决策差异的失败、无独立证据的 Agent 观察、用户禁止泛化、个人级或跨项目目标不进入工作流，也不输出回执。
- 发送最终回复前检查本轮全部独立决策。发现一条候选不能替代其他候选。

## 模式

- `safe`：任何来源的可沉淀候选通过安全门后自动写入，不追加确认门。
- `review`：只提案；用户批准精确规则和目标位置后重新通过安全门再写入。
- `off`：关闭主动发现和持久化，只响应手动调用 `$agent-evolve`。
- 用户当前禁止写入、限制为一次性或要求文件不变时，该边界优先于 `safe`。

## 渐进路由

1. 自动入口先完成交付分离和所有权排除；决定属于当前有效权威内的遵循、执行纠正或原位修改时立即停止，不读取 reference、不输出回执。剩余输入有候选时读取 `references/workflow.md`，无候选时停止。手动入口直接读取 `references/workflow.md`。
2. 工作流完成候选、模式、落点和冲突判断前，禁止读取 `references/validation.md`。
3. 进入安全验证后读取 `references/validation.md`；正常状态和默认回执以该文件为准。
4. 只有未解决冲突或处理失败时，才读取 `references/failure-receipt.md` 生成展开回执；其他分支禁止读取。

## 全局边界

- 只在当前项目的主会话中自动运行，只处理当前项目规则。
- 禁止写入用户级、全局、其他项目或第三方规则源。
- 先完成当前任务，再处理不阻塞任务的候选。
- 候选与有效规则冲突时，在修改会选边的实现或测试前进入工作流并停止选边。
- 禁止扫描完整对话做会后复盘、使用模型记忆补造反馈、创建反馈收集箱、自动提交 git commit 或启动额外 agent 轮次。
