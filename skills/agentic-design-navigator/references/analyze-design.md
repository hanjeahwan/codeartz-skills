# B 模式：意图成形

## 目标

将暂定目标方向操作化为足以比较设计选择的设计意图基线。将九个设计维度作为后台导航地图，按当前判断需要调用。

## 输入

- 输入必须包含 A 模式形成的暂定目标方向，或用户直接给出的等价目标方向。
- 缺少暂定目标方向时，返回 A 模式。

## 层级区分

- 目标方向描述希望发生的变化及其意义。
- 设计意图描述设计要支持的主体、情境、能力、边界和成功标准。
- 方案机制描述实现设计意图的可选路径。
- 功能和实现只属于方案机制，禁止反向替换目标方向或设计意图。

## 九维导航地图

- 只分析会改变当前判断的维度。
- 不适用的维度标记为“不适用”。
- 缺少依据的内容标记为“未知”或“待验证”。

| 维度       | 导航判断                                                         |
| ---------- | ---------------------------------------------------------------- |
| 主体       | 谁直接行动、间接受影响、承担成本、无法使用或没有发言权           |
| 情境       | 问题在什么任务、关系、环境和限制下发生                           |
| 目的       | 主体真正想改变、避免或保持什么                                   |
| 行动可能   | 设计使什么行为更容易、更安全、更明显或更困难                     |
| 转化条件   | 理解、意愿、权限、能力、环境、时机、成本和协作是否允许行动发生   |
| 结果       | 可能出现哪些直接、间接、短期、长期和意外结果                     |
| 价值与代价 | 谁获得什么价值，谁承担时间、认知、隐私、安全、依赖或额外劳动成本 |
| 公平性     | 收益、付费、风险、发言权、使用权和退出权如何分配                 |
| 验证       | 哪些是事实、推断或假设，什么证据可以支持或推翻关键假设           |

分析因果关系时使用以下链条：

```text
设计要素
→ 改变信息、行为、权限或环境
→ 提供行动可能
→ 在转化条件下被实际采用
→ 产生直接结果
→ 形成长期价值或代价
```

## 设计意图基线

基线只保留影响方向选择的内容：

- 暂定目标方向。
- 关键主体与情境。
- 希望支持的行动或能力。
- 可观察的成功变化。
- 有效边界与不可接受结果。
- 关键取舍。
- 可能反转方向的假设及其状态。
- 最小验证方式。

- 没有依据的字段使用“未知”“待验证”或“不适用”。
- 禁止为了形成完整基线补造内容。

## 前台响应

- 普通回合只展示本轮新增、修正、冲突或会改变方向的内容。
- 用户要求总结或准备进入 C 模式时，展示完整设计意图基线供用户校准。
- 成功标准必须描述主体、行为、关系或环境的可观察变化。
- 禁止把功能交付、上线或使用次数单独作为成功标准。

## 完成判据

- 检查项：设计意图基线足以判断一个方向是否服务目标。
- 检查项：可能反转方向的未知已经澄清，或明确保留为待验证假设。
- 检查项：成功变化与方案机制已经区分。
- 检查项：用户已经看到设计意图基线，并能据此修正方向。
- 放行条件：全部检查项满足。
- 失败处理：继续在 B 模式分析当前关键维度。
- 返回条件：分析暴露会改变问题定义的竞争理解。
- 返回处理：回到 A 模式。
