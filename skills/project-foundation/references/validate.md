# 第二阶段：验证草稿

## 目标

独立验证当前草稿是否满足请求用户批准的条件。验证不重新诊断项目，不替用户裁决，不修改草稿。

## 确定性验证

- 运行 `node <skill-root>/scripts/validate-draft.ts .codeartz/project-foundation/draft.md`。
- 脚本退出码为 `0` 时继续语义验证。
- 脚本退出码为 `1` 时报告每个失败检查并返回第一阶段。
- 脚本退出码为 `2` 时报告读取或解析错误并返回第一阶段。
- 禁止跳过失败检查。
- 禁止为了通过验证而在本阶段修改草稿。
- 禁止复制、改写或生成验证器替代品。
- 执行权限不可用时报告阻塞，不声称验证通过。

## 语义验证

- 将每条建议正文拆成可独立验证的原子陈述。
- 检查每个原子陈述的时间、范围、行为、关系、职责与规范强度是否有直接证据。
- 检查存在证据是否被扩大为实际使用、采用关系、行为历史、设计意图、长期职责或规范强度。
- 检查影响目标、正文或合并范围的选择是否全部进入待裁决。
- 检查批准请求是否只允许批准或拒绝唯一最终内容。
- 检查同一证据在不同区块中的结论强度是否一致。
- 任一检查失败时引用草稿原文与反证并返回第一阶段。

## 草稿身份

- 语义验证完成后再次运行确定性验证脚本。
- 第二次验证失败时返回第一阶段。
- 两次 `draftSha256` 不一致时返回第一阶段。
- 两次验证通过且 hash 一致时，使用该 hash 请求用户批准。
- 用户要求修改草稿时返回第一阶段。
- 用户批准的 hash 与当前草稿 hash 不一致时重新验证。

## 输出

- 输出确定性验证状态。
- 输出语义验证状态。
- 输出已验证的 `draftSha256`。
- 验证失败时逐项输出失败原因与证据。
- 验证通过时请求用户批准该 hash 对应的唯一最终内容。
- 禁止创建持久化验证状态文件。
- 禁止启动新的 Agent 会话。
