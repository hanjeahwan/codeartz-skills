# 任务合同产物手册

## 目标

- 维护唯一的 `.codeartz/<topic>/tasks.md`，使新 Agent 能复核状态并继续执行。
- 把目标来源、计划基线、任务图、完成证据和组合关卡保持在一个易读入口。
- 本阶段不修改实现、不执行任务、不提交 Git。

## 产物路径

- 用户可以指定 `<topic>`；否则从点名的功能、issue、变更、模块、路由或主要对象生成。
- `<topic>` 使用小写 kebab-case，只保留安全文件名字符；路径解析后必须位于 `.codeartz/` 内。
- `<topic>` 无法安全确定时停止写入，报告需要确认的主题名称。
- 固定写入 `.codeartz/<topic>/tasks.md`。用户不能改变 `.codeartz/` 根目录或 `tasks.md` 文件名。
- 已有 `tasks.md` 时先读取再更新同一文件；禁止创建日期副本或第二份有效合同。
- 用户明确要求只在对话返回时不写入项目。
- 项目不可写或写入失败时，在对话返回完整合同并报告实际状态。

## 写入关卡

- 只有 `SKILL.md` 的输入、任务切分和依赖完成条件全部通过后，才创建正式任务合同。
- 输入不足、目标冲突或关键证据失效时，不创建看似可开工的 `tasks.md`；已有文件保持不变。
- 工作区已有改动与任务写集重叠时，把重叠写为阻塞，不回退或吞并现有改动。
- 写入前加载 `assets/tasks-template.md`，删除不适用的条件章节和模板说明。

## 内容合同

### 执行基线

- `目标来源`：引用用户目标、spec、`boundary.md` 或其他实际输入；不复制上游完整内容。
- `计划基线`：记录生成或复核时的 HEAD；非 Git 仓库写“不适用”。
- `工作区状态`：记录 clean、dirty 或非 Git 仓库的“不适用”。
- `失效触发`：只列会改变任务切分、顺序、边界或验证方式的变化。

### 任务索引

- 索引是任务状态、依赖和执行边界的唯一权威位置。
- 状态只使用“待执行”“已完成”“阻塞”。禁止使用崩溃后无法证实的“进行中”。
- 执行只使用 `serial` 或 `parallel-safe`；依赖不存在时写“无”。
- 任务保持少量；不为生成索引复制第二份任务图。

### 任务卡片

- 每张卡片只包含：结果、范围、入口、验收、验证和提交。
- 卡片不重复状态、依赖或执行边界。
- 已完成任务追加“完成证据”，只记录当前 checkout 可定位的 commit 和实际验证命令及结果。
- 阻塞任务追加“阻塞”，记录原因和可判断的解锁条件。
- 待执行任务不保留空的“完成证据”或“阻塞”章节。

### 组合关卡

- 只保留“整体测试”和“提交范围审查交接”。
- 关卡状态只使用“待执行”“已完成”“阻塞”。
- 没有实际运行证据时不得写“已完成”。

## 恢复与更新

- 先读取 `tasks.md`、目标来源和当前 checkout，再判断任务状态。
- 文本状态不能单独证明完成；commit 或验证证据缺失、不可定位或与当前 checkout 不兼容时，把对应任务标为“阻塞”，并写明恢复缺口。
- 目标边界、关键入口、依赖或任务基线变化会改变任务图时，停止领取受影响任务，重新拆分后更新同一文件。
- 只更新证据支持的状态和受影响任务；不因接手请求把“待执行”升级为“已完成”。
- 所有依赖任务完成且证据可核对后，才允许领取下游任务或运行组合关卡。
- 并行 worker 不更新共享 `tasks.md`；由协调者收集证据后串行更新。

## 对话输出

- 成功写入时，只返回合同路径、任务数量、首个可领取任务或阻塞摘要。
- 恢复时返回已证实状态、失效证据和下一项安全动作。
- 不把任务合同生成或恢复说成实现、测试、提交或审查已经完成。
