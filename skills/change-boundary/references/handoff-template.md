# 交接文件输出模板

## 使用规则

- 写入 `.swe/<topic>/handoff.md` 时，使用此模板。
- 只在用户确认风险、裁决待裁决点或确认无待裁决点，并同意方案后使用。
- 信息不足时，不填充猜测；写明缺口、未知、未记录或待补证据。
- 交接文件不能改写方案结论、写入新的方案取舍或包含文件级任务清单。

```markdown
## 交接文件

### 已采纳方案

- 方案文件：[.swe/<topic>/change-boundary.md]
- 用户确认：[用户接受方案、理解风险、裁决待裁决点或确认无待裁决点的原话或会话定位。]
- 当前证据等级：[L3。]
- 不能声称：[仍不能声称已实现、已验证或生产安全的内容。]

### 证据状态

- 项目位置：[仓库、worktree、cwd 或用户点名路径。]
- 当前分支：[能获取时填写；不能获取时写“未记录”。]
- 工作区状态：[能获取时填写是否有未提交变更；不能获取时写“未记录”。]
- 过期触发：[分支变化、相关文件变更、方案文件更新或依赖版本变化时，哪些证据必须重新复核。]

### 任务入口

- 原始请求：[用户需求、PRD、issue、会话或截图来源。]
- 目标行为：[一句话目标。]
- 禁止扩散：[不能改的路径、分区、合同或外部接口。]
- 采用边界：[交接文件继承的适用分区、不适用分区和保持原行为。]

### 探索链路

- 非结构化资料：[已经读取和抽取的资料。]
- 资料假设：[已形成的假设。]
- 代码事实：[已确认的当前系统行为。]
- 语义边界：[已建立的分区和边界。]
- 系统现实：[资料和代码合并后的真实状态。]
- 根因判断：[如果适用，写根因；否则写“不适用”。]
- 目标合同：[采用方案依赖的合同摘要。]
- 记录限制：[只写已读资料、证据定位、结论和缺口；不写隐藏推理过程。]

### 证据索引

- [文件 / 符号 / 测试 / 配置 / 命令]：
  - 说明：[该证据实际说明什么。]
  - 状态：[支持 / 反证 / 限制 / 冲突 / 未知。]
  - 影响分区：[它影响哪个分区和窄语义行为。]

### 已查路径

- [搜索词 / 文件 / 测试 / 调用链]：
  - 结论：[查到什么或没有查到什么。]
  - 停止原因：[为什么不用继续查。]

### 未查路径

- [搜索词 / 文件 / 测试 / 调用链]：
  - 未查原因：[为什么当前没查。]
  - 触发条件：[什么时候必须补查。]

### 分区授权

- 覆盖声明：[是否覆盖所有已知分区；没有覆盖时，新会话必须先补哪些分区。]
- [分区]：[已授权 / 禁止 / 保持原行为 / 未知 / 冲突 / 不适用]
  - 证据：[关键证据。]
  - 开工处理：[实现时允许、禁止、保持、先补证据或先回问。]
  - 复核触发：[哪些文件、分支、依赖、配置或资料变化会让该授权过期。]

### 开工规则

- 先复核：[新会话开工前必须打开的关键文件、符号、测试或配置。]
- 不要重复：[已确认且无需重复探索的路径，除非代码已变。]
- 先补缺口：[实现前必须补的未知或外部事实。]
- 保持边界：[实现时不能越过的归属方、状态、数据契约、副作用和兼容约束。]
- 完成声明限制：[没有哪些验证结果时，不能声称完成。]

### 开工校验

- 开工前读取：[方案文件、先复核列表和会过期的关键证据。]
- 先补证据：[信息缺失、未记录或已过期的证据；补完前不得改相关分区。]
- 禁止越界：[未知、冲突、禁止、不适用和保持原行为分区不得套用目标行为。]
- 跳过重复探索：[只跳过未过期且已写明停止原因的路径；代码、分支、依赖或资料变化时重新复核。]
- 完成声明：[没有实现结果和验证结果时，只能说明“已交接”，不能声称完成、已验证或可上线。]
```
