# 交接文件阶段手册

## 阶段职责

交接文件阶段负责生成 `.swe/<topic>/handoff.md`。

生成前必须满足：

- 待裁决点已处理或明确无待裁决点。
- 风险已在方案中展示。
- 用户同意采用当前方案。

本阶段复用已探索链路，不改写方案结论。

方案结论、裁决、风险、不能声称和证据等级以方案文件为准。

交接文件只写接力导航、证据索引、已查路径、未查路径、分区授权的开工处理和开工规则。

## 写入前检查

### 交接文件质量门槛

- 必备材料：
  - 缺少以下任一项时，不生成交接文件：方案文件、用户同意或生成请求、证据索引、分区授权或开工规则。
- 证据索引范围：
  - 只包含可定位证据。
  - 不能用模糊表述替代证据定位。
  - 模糊表述包括“之前看过”“代码里有”“应该是”。
  - 证据定位必须写路径、符号、测试、配置或命令。
- 证据索引粒度：
  - 优先定位到文件 + 符号、测试名、配置名、命令或关键行号。
  - 只能写文件级路径时，说明为什么没有更细定位。
  - 写明新会话开工时要复核的符号或搜索词。
- 新会话可接手性：
  - 交接文件必须让新会话知道先读哪些文件和为什么读。
  - 只列文件名、不写证据说明时，交接文件不合格。
- 已查和未查路径：
  - 已查路径必须说明停止原因。
  - 未查路径必须说明触发条件。
  - 两者缺失时，新会话会重复探索或漏掉风险。
- 方案一致性：
  - 交接文件不得重新裁定方案结论、风险、待裁决点、不能声称或证据等级。
  - 需要引用这些内容时，指向方案文件。
  - 本文件与方案文件不一致时，先更新方案文件或输出冲突清单。
- 裁决回写：
  - 用户裁决待裁决点后，先更新方案文件。
  - 回写受影响的范围和授权、目标合同、风险和待裁决、不能声称。
  - 回写后再判断是否可以生成交接文件。
- 探索链路来源：
  - 交接文件不能补造当前会话没有记录的探索链路。
  - 信息缺失时写“未记录”。
  - 把补证据动作放入开工规则。

## 产物规则

### 交接文件产物

生成第二份产物前必须满足：

- 待裁决点已处理或明确无待裁决点。
- 风险已在方案中展示。
- 用户同意采用当前方案。

满足后生成交接文件。

交接文件让没有上下文的新会话复用已探索链路。

目标是避免重复探索同一批资料和代码。

- 默认路径为 `.swe/<topic>/handoff.md`。
- 用户明确要求其他路径时，使用用户路径，但仍和方案文件分开。
- 生成前读取方案文件：
  - 先读取对应的 `.swe/<topic>/change-boundary.md`。
  - 确认待裁决点已处理或明确为无。
  - 确认风险已在方案中展示。
  - 确认用户同意采用当前方案。
- 用户裁决待裁决点时：
  - 先把裁决回写到方案文件。
  - 待裁决清空后，在对话里推荐下一步生成交接文件。
  - 不要求用户重复说“接受方案”或“理解风险”。
- 用户同意采用当前方案：
  - 用户明确要求生成 handoff 时，视为同意采用当前已裁决方案。
  - 用户只问是否该生成 handoff 时，不视为同意采用方案。
- 阻塞生成条件：
  - 方案未展示风险，或用户要求继续调整风险处理。
  - 方案仍有未裁决待裁决点。
  - 方案仍有冲突分区。
  - 方案仍有阻塞方案开工的未知分区。
  - 用户未同意采用方案。
  - 命中以上任一项时，只输出缺口清单或下一步建议，不生成交接文件。
- 已有交接文件时：
  - 先读取现有内容。
  - 再决定更新、覆盖或另存。
  - 不要静默丢失已有交接信息。
- 交接文件内容：
  - 记录已探索链路、证据索引、已查路径、未查路径、分区授权和开工规则。
  - 已探索链路只写已读资料、证据定位、结论和缺口。
  - 不写隐藏推理过程。
  - 禁止改写方案结论。
  - 禁止写文件级任务清单。
  - 禁止声称已经实现或验证。
- 新会话使用交接文件时：
  - 先复核交接文件点名的关键证据。
  - 再补缺口。
  - 不要默认重复完整探索。

### 交接文件生成步骤

生成交接文件时按以下顺序执行：

1. 定位主题目录。
   - 优先使用用户点名路径。
   - 否则使用当前方案文件所在的 `.swe/<topic>/`。
2. 读取 `.swe/<topic>/change-boundary.md`。
   - 找不到方案文件时，输出缺失路径和需要先生成方案文件的动作。
   - 不从记忆补写交接文件。
3. 处理用户裁决和生成条件。
   - 用户已裁决待裁决点时，先回写方案文件。
   - 确认待裁决点已处理或明确为无。
   - 确认风险已在方案中展示。
   - 确认用户同意采用当前方案。
   - 确认方案文件已记录已裁决的方案边界。
   - 用户明确要求生成 handoff 时，把本轮请求作为生成条件。
   - 不把 handoff 请求原话写入交接文件正文。
   - 条件缺失时，只输出缺口清单或下一步建议。
4. 从已发生的探索过程和方案文件提取内容。
   - 只能写入已经读取、已经引用或已经明确记录的内容。
   - 内容包括资料、代码事实、搜索路径和反证。
   - 缺失内容标为“未记录”或“待补证据”。
5. 建立证据索引。
   - 每条证据写明定位、说明、状态、影响分区和开工复核要求。
   - 没有定位的证据不能作为代码事实写入。
6. 区分已查路径和未查路径。
   - 已查路径写结论和停止原因。
   - 未查路径写未查原因和触发补查条件。
7. 写开工规则。
   - 只写新会话开工前必须复核什么。
   - 写哪些探索不要重复。
   - 写哪些缺口先补。
   - 写哪些边界不能越过。
8. 写入 `.swe/<topic>/handoff.md`。
   - 对话回复只给：交接文件路径、采用方案路径、必须先复核的证据和必须遵守的方案文件限制。

## 输出模板

### 交接文件输出模板

#### 使用规则

- 写入 `.swe/<topic>/handoff.md` 时，使用此模板。
- 使用前置：
  - 待裁决点已处理或明确无待裁决点。
  - 风险已在方案中展示。
  - 用户同意方案。
- 信息不足时：
  - 不填充猜测。
  - 写明缺口、未知、未记录或待补证据。
- 交接文件不能：
  - 改写方案结论。
  - 写入新的方案取舍。
  - 包含文件级任务清单。

```markdown
## 交接文件

### 方案入口

- 方案文件：[.swe/<topic>/change-boundary.md]
- 读取顺序：先读方案文件，再读本交接文件。
- 边界来源：方案结论、裁决、风险、不能声称和证据等级以方案文件为准。
- 冲突处理：本文件与方案文件冲突时，停止开工，先复核方案文件。

### 任务入口

- 原始请求：[用户需求、PRD、issue、会话或截图来源。]
- 目标行为：[一句话目标。]
- 禁止扩散：[不能改的路径、分区、合同或外部接口。]
- 采用边界：[交接文件继承的适用分区、不适用分区和保持原行为。]

### 探索链路

- 非结构化资料：[已经读取和抽取的资料。]
- 资料假设：[已形成的假设。]
- 代码事实：[已确认的当前系统行为。]
- 语义边界：[已建立的分区和边界。]
- 系统现实：[资料和代码合并后的真实状态。]
- 根因判断：[如果适用，写根因；否则写“不适用”。]
- 目标合同：[采用方案依赖的合同摘要。]
- 记录限制：[只写已读资料、证据定位、结论和缺口；不写隐藏推理过程。]

### 证据索引

- [文件 + 符号 / 测试名 / 配置名 / 命令 / 关键行号]：
  - 说明：[该证据实际说明什么。]
  - 状态：[支持 / 反证 / 限制 / 冲突 / 未知。]
  - 影响分区：[它影响哪个分区和窄语义行为。]
  - 开工复核：[新会话开工时要复核的符号、搜索词或行号；只能文件级定位时写原因。]

### 已查路径

- [搜索词 / 文件 / 测试 / 调用链]：
  - 结论：[查到什么或没有查到什么。]
  - 停止原因：[为什么不用继续查。]

### 未查路径

- [搜索词 / 文件 / 测试 / 调用链]：
  - 未查原因：[为什么当前没查。]
  - 触发条件：[什么时候必须补查。]

### 分区授权

- 授权来源：以方案文件为准；本节只记录开工处理和复核触发。
- 覆盖声明：[是否覆盖所有已知分区；没有覆盖时，新会话必须先补哪些分区。]
- [分区]：[已授权 / 禁止 / 保持原行为 / 未知 / 冲突 / 不适用]
  - 证据：[关键证据。]
  - 开工处理：[实现时允许、禁止、保持、先补证据或先回问。]
  - 复核触发：[哪些文件、分支、依赖、配置或资料变化会让该授权过期。]

### 开工规则

- 先复核：[新会话开工前必须打开的关键文件、符号、测试或配置。]
- 不要重复：[已确认且无需重复探索的路径，除非代码已变。]
- 先补缺口：[实现前必须补的未知或外部事实。]
- 保持边界：[实现时不能越过的归属方、状态、数据契约、副作用和兼容约束。]
- 过期触发：[方案文件、相关代码、分支、依赖、配置或资料变化时，哪些证据必须重新复核。]
- 完成声明限制：[没有哪些验证结果时，不能声称完成。]

### 开工校验

- 开工前读取：[方案文件、先复核列表和会过期的关键证据。]
- 先补证据：[信息缺失、未记录或已过期的证据；补完前不得改相关分区。]
- 禁止越界：[未知、冲突、禁止、不适用和保持原行为分区不得套用目标行为。]
- 跳过重复探索：[只跳过未过期且已写明停止原因的路径；代码、分支、依赖或资料变化时重新复核。]
- 完成声明：[没有实现结果和验证结果时，只能说明“已交接”，不能声称完成、已验证或可上线。]
```
