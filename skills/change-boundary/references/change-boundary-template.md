# 方案输出模板

## 使用规则

- 写入 `.swe/<topic>/change-boundary.md` 时，使用此模板。
- 只在方案关卡通过且证据等级达到 L3 时写入项目文件。
- 关卡失败时，不使用此模板写文件；只在对话里输出缺口、被阻止的结论和下一步证据动作。
- 字段信息不足时，不填充猜测；写明缺口、未知或非阻塞待补证据。
- 方案文件只写方案内容。不要写入交接文件内容、完整探索链路、已查路径或未查路径。
- 没有会改变执行结果的被拒方案时，写“不适用”，不要为了填模板补造备选。

```markdown
## 方案

### 推荐结论

[一句话说明推荐怎么做。]

### 范围和授权

- 适用分区：[哪些分区、入口、模块、对象、状态或用户路径应用该方案。]
- 不适用分区：[哪些反证、未知、冲突、不适用或保持原行为分区不应用该方案。]
- 覆盖声明：[是否覆盖所有已知分区；未覆盖时写原因和处理方式。]
- 授权明细：[分区] - [已授权 / 禁止 / 保持原行为 / 未知 / 冲突 / 不适用] - [证据和处理方式。]
- 反证和未知：[为什么不能把目标行为套用到这些分区。]

### 依据

- 资料假设：[来自需求、PRD、issue、会话或旧方案的关键假设。]
- 代码事实：[来自入口、调用链、类型、状态、测试、配置或副作用归属的事实。]
- 外部事实：[来自官方文档、源码、协议、平台限制或固定版本依赖的约束；没有则写“不适用”。]
- 系统现实：[资料和代码合并后，当前系统真实表现。]
- 根因判断：[如果是故障、事故或异常任务，写根因；否则写“不适用”。]
- 证据等级：[L3，以及不能声称的内容。]

### 目标合同

- 目标行为：[新增、修改、删除或保持什么行为。]
- 输入输出：[输入、输出、状态变化。]
- 副作用：[写数据、发请求、发通知、缓存、权限、异步任务等影响。]
- 失败行为：[失败、跳过、fallback、直接通过或报错方式。]
- 兼容约束：[旧路径、旧数据、旧 API、旧 UI 或旧行为如何保持。]

### 变更边界

- [建议改动的归属方、边界、状态归属或抽象层。]
- [为什么该边界优先于相邻路径。]
- [如何避免把行为扩散到未授权分区。]
- [方案需要保留的实现约束、验证前置或发布/回滚/观测前置。]

### 被拒方案

- [方案名或“不适用”]：[被拒方案或不适用原因。]
  故障模式：[会误改什么、漏掉什么、破坏什么边界。]

### 风险和待裁决

- 风险：[已授权方案内仍可能爆发的问题；不要把未授权分区写成风险。]
- 待裁决：[未授权同入口变体、范围、行为或取舍；被上层目标覆盖但没有明确排除、且缺少目标行为依赖机制的变体；没有则写“不适用”。]
- 证据停止点：[哪些证据因为重复、范围过大、不可访问或不改变结论而停止；没有则写“不适用”。]
- 待补证据：[不阻塞本方案结论，但实现、验证或发布前仍需补的代码事实或外部事实；没有则写“不适用”。]
- 不能声称：[由于证据等级或关卡限制，本轮不能声称的结论。]
```
