## Skill 与手册关系

- `SKILL.md` 只放触发条件、阶段路由、全局边界和禁止动作。
- 阶段手册放对应阶段的执行规则、检查项、产物规则和输出模板。
- 同一条规则只保留一个权威位置；移动到手册后，从 `SKILL.md` 删除详细内容。
- 禁止复制代替移动，避免双份规则和后续漂移。
- 多个阶段手册保持结构一致，内容只服务各自阶段。

## 专注机制

- agent 执行某阶段时，只读取该阶段手册，不预读后续阶段手册。
- 阶段切换必须由明确条件触发；条件未满足时，不进入下一阶段产物。
- 编辑 skill 时，先判断内容属于入口路由还是阶段执行。
- 修改手册结构时，检查同类手册层级；修改规则内容时，只改权威位置。

## 自检

- **结构一致**：内容层级、标题格式、步骤顺序保持统一。
- **顺序合理**：先写目标，再写执行规则，最后写限制与输出要求。
- **指令聚焦**：每条指令只表达一个明确动作，避免一条里混多个要求。
- **避免重复**：删除语义相近、作用重叠的内容，防止 agent 不知道优先执行哪条。
- **优先级清楚**：当规则可能冲突时，明确哪个指令优先。
- **内容可执行**：避免抽象描述，多使用可直接执行的动词，例如“检查”“生成”“禁止”“保留”。
- **边界明确**：写清楚什么时候做、什么时候不做、遇到不确定时怎么处理。
- **输出明确**：指定输出格式、长度、语气、是否需要引用或列表。
- **示例必要**：只保留能消除歧义的示例，避免示例过多干扰主规则。
- **无隐含假设**：不要让 agent 猜目标、对象、格式或成功标准。
